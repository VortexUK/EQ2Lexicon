# Tests + Coverage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Burn down all 86 findings from the tests + coverage audit (`docs/superpowers/specs/2026-05-30-tests-audit.md`) in five sub-phases — broken/lying tests + critical untested code first, then test infrastructure (factories + fixture hoists), then test file splits, then adoption + duplicates removal, then the bulk coverage-gap closing, and finally polish. Target: lift in-scope coverage from 66 % → 80 %.

**Architecture:** Phase 1 (P0) fixes the two "lying test" / broken-assertion issues and ships the highest-priority untested production-code tests (`notifications.py`, `census/client.py`'s HTTP backbone, `web/routes/item.py` search + detail). It also resolves the two dead-helper P0 questions (`_shared.py`, `officer_gate.py`) by DELETING them — both shipped during Phase 2a of the backend-cleanliness work, neither was ever adopted, and the audit explicitly flags them as cleanup candidates. Phase 2a stands up the canonical test infrastructure in `tests/fixtures/users.py` (the shared `make_fake_admin()` / `make_fake_user()` factories that 10+ test files redeclare), hoists the per-test parses-DB fixture out of `tests/parses/conftest.py` so `tests/web/` can use it too, and adds the autouse logging-state-snapshot fixture that fixes the global-state leak. Phase 2b splits the 1,354-line `test_parses.py` into focused siblings mirroring `web/routes/parses/`, and extracts the 400-line `_seed_act_db` blob in `tests/parses/conftest.py` into a JSON fixture file. Phase 2c is the mechanical migration to the Phase 2a helpers + the conftest tmp-dir race fix. Phase 3 is the bulk coverage-gap-closing — 5 sibling test files for `census/client.py` (the single biggest gap), plus focused test files for `item.py`, `item_watch.py`, `character/upgrades.py`, `db/users.py`, `recipes_db.py`, `raids_act_db.py`. Phase 4 is polish — naming consistency, parametrize sweeps, weak-assertion strengthening.

**Tech Stack:** Python 3.12+, pytest 8.x with `pytest-asyncio` mode=auto, FastAPI test client via `httpx.AsyncClient(transport=ASGITransport(app=app))`. `uv` for dependency + script execution (no PATH prefix; `uv run …` works straight from PowerShell — see memory `uv-on-path-no-prefix.md`). Coverage runs are gated by `pytest-cov`, invoked as `uv run --with pytest-cov pytest …`. The `tests/` tree mirrors the production layout: `tests/web/`, `tests/census/`, `tests/parses/`; `tests/conftest.py` sets `DB_*_PATH` env vars at `pytest_configure` time and pre-builds the schemas.

Spec: `docs/superpowers/specs/2026-05-30-tests-audit.md` (86 findings: TEST-P0 = 2, TEST-P1 = 14, TEST-P2 = 32, COV-P0 = 7, COV-P1 = 18, COV-P2 = 13). Each task references the spec's TEST-NNN / COV-NNN IDs. Per the audit's bucket counts the spec lists 2 P0 TEST findings but none are individually tagged P0 in the body; this plan assigns the two "lying test / broken assertion" items (TEST-014 + TEST-022 — which both ship false-positive assertions per the audit) to Phase 1 alongside the COV-P0 work.

## Conventions for every task

1. **Verification per task** runs `uv` directly — no PATH prefix needed (memory `uv-on-path-no-prefix.md`):
   - `uv run ruff format <touched-files>` — auto-fmt; non-fatal.
   - `uv run ruff check <touched-files>` — lint must pass clean.
   - `uv run pytest <scope> -v -x` — bail on first failure. Scope per task: `tests/web/<file>.py` for a single new file, `tests/web/` or `tests/census/` for broader changes, `tests/` for cross-cutting work.
   - For **every COV-NNN task** additionally run:
     `uv run --with pytest-cov pytest <scope> --cov=<production-path> --cov-report=term-missing -q`
     and assert the listed target % is met. Target % is named in each task; failure to meet it means more scenarios are needed before the task ships.
2. **NO commits inside tasks.** Per memory [[hold-commits-on-visual-work]] — even though tests aren't visual, the convention is kept so the user can spot-check each phase before it ships. Every change is held for user review. Each phase ends with a single **commit-checkpoint task** the user runs after approval.
3. **Stage ONLY the named files at each checkpoint.** Never `git add -A` — the user may have unrelated WIP. The commit-checkpoint task explicitly lists the files to add and any to delete (`git rm`).
4. **Branch:** the user works on `main`. Each phase commit can push direct to main via `git push origin main` (the pre-push hook in `.githooks/pre-push` runs ruff/pyright/pytest; this plan's per-task verification ensures pre-push passes).
5. **No `--no-verify`.** If the pre-push hook fires on the commit step, fix the failure — don't bypass.
6. **Behavioural scenarios, not line-coverage chasing.** Every new test should encode a behaviour the reader can name in one sentence. If a test's only justification is "it lifts coverage of line N", refactor until it expresses a real contract.
7. **Test naming.** New tests follow the BDD-style `test_<scope>_<action>_<state>` convention (e.g. `test_signature_rejected_when_body_tampered`, `test_search_dedupes_by_canonical_name`). This is the recommendation in TEST-019; codified here for Phase 1+ work.
8. **Typed fakes.** Test fakes returning a user-shaped dict MUST be typed `SessionUser` (`web/lib/session_user.py`) so pyright catches drift (TEST-031). The Phase 2a factory takes care of this for callers that adopt it.
9. **No global-state mutation.** Tests that exercise module-level caches (`web.routes.supporters._cache`, `census.spells_db.find_by_crc.cache_*`, `logging.getLogger().level`) MUST be wrapped by an autouse isolation fixture. Phase 2a creates the canonical fixtures for the three known cases.
10. **No `assert isinstance(_, list)`-only assertions.** Every test asserts the actual content shape, not the framework's guarantee. If the only thing under test is a framework behaviour (e.g. FastAPI 422 on a missing query param), the test is deleted (TEST-022).
11. **Pin exact status codes.** No `assert r.status_code in (401, 403)` — pick one. Unauthenticated → 401; authenticated but not authorised → 403 (TEST-023).
12. **Subset over equality.** Prefer `assert body["ok"] is True` over `assert body == {"ok": True}` when the goal is "the call succeeded"; reserve full-equality for explicit shape-pinning tests (TEST-036).
13. **Async tests.** Project uses `pytest-asyncio` mode=auto (see `pyproject.toml`); the `@pytest.mark.asyncio` marker is implicit. New tests using `async def` work without the marker.
14. **HTTP-layer tests.** Use the global `app` fixture from `tests/conftest.py` and the standard `httpx.AsyncClient(transport=ASGITransport(app=app), base_url="http://test")` pattern. Don't reintroduce local `app` fixture overrides unless the test genuinely needs a custom `session_secret` (only `test_auth.py` qualifies — TEST-002 / TEST-003).
15. **DB-layer tests.** Use the per-test temp DB path provided by `parses_db_path` (Phase 2a hoist) or `_TEST_DB_DIR` from `tests/conftest.py`. In-memory SQLite (`":memory:"`) is preferred for pure-helper tests that don't need to persist across calls.
16. **Coverage measurement.** When a task says "target X % coverage on file Y", that's the threshold the `--cov-report=term-missing` output must show after the new tests run. Use `--cov=<dotted.module.path>` not a filesystem path so coverage measures the in-import-paths code.

```python
Scoped autouse so every test gets it — the snapshot/restore is cheap

from __future__ import annotations

import logging
from typing import Generator

import pytest

_LOGGER_NAMES = (
    "",  # root
    "discord",
    "uvicorn.access",
    "aiohttp.access",
```

## Task 2c.6: Fix `tests/conftest.py` tmp-dir race + dead constant (TEST-039, TEST-048)

**Files:** `tests/conftest.py`

The conftest wipes `_TEST_DB_DIR` at module-import time (`tests/conftest.py:34-37`). If two pytest processes import this conftest simultaneously (`pytest -n auto`), they race on `rmtree` + `mkdir`. Once parallel test execution is enabled this becomes a flaky-test source.

- [ ] **Step 1: Move the wipe into a session-scoped autouse fixture**

Before (current lines 34-37):
```python
_TEST_DB_DIR = Path(tempfile.gettempdir()) / "eq2lexicon-pytest"
if _TEST_DB_DIR.exists():
    shutil.rmtree(_TEST_DB_DIR, ignore_errors=True)
_TEST_DB_DIR.mkdir(parents=True)
```

After:
```python
# Per-process suffix to make parallel pytest workers race-free.
_PROC_SUFFIX = f"{os.getpid()}"
_TEST_DB_DIR = Path(tempfile.gettempdir()) / f"eq2lexicon-pytest-{_PROC_SUFFIX}"
_TEST_DB_DIR.mkdir(parents=True, exist_ok=True)


@pytest.fixture(scope="session", autouse=True)
def _tmp_db_dir_isolation() -> Generator[Path, None, None]:
    """Wipe + recreate the tmp DB dir at session start AND teardown.

    Per-process suffix means parallel pytest-xdist workers don't race
    on rmtree (TEST-039).
    """
    # Already created above (so pytest_configure can read it). Wipe contents.
    if _TEST_DB_DIR.exists():
        for entry in _TEST_DB_DIR.iterdir():
            if entry.is_file():
                entry.unlink()
            elif entry.is_dir():
                shutil.rmtree(entry, ignore_errors=True)
    try:
        yield _TEST_DB_DIR
    finally:
```
